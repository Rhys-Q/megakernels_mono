# 核心概念入门

> 这篇文档解释理解 Megakernel 所需的基础知识。如果你已经熟悉 GPU 编程和 Transformer，可以跳过。

---

## 1. GPU 是什么？为什么适合跑 AI？

普通 CPU 有几十个核，擅长处理复杂的串行逻辑。  
GPU 有几千个核（RTX 3090 有 10496 个 CUDA 核），擅长同时做大量简单的数学运算。

运行语言模型的核心操作是**矩阵乘法**（把一行数字乘以一个矩阵）。这种操作天然适合并行：矩阵的每一行可以同时计算，互不干扰。

### GPU 上的计算单位

- **Thread（线程）**：最小的执行单元，做一个具体的数学操作
- **Warp（束）**：32 个线程的组合，GPU 以 warp 为单位调度执行
- **Block（块）**：多个 warp 组成一个 block，block 内可以用共享内存通信
- **Grid（网格）**：所有 block 的集合，对应一次 kernel 调用

RTX 3090 有 82 个 SM（流式多处理器），每个 SM 可以同时驻留多个 block。

## 2. CUDA Kernel 是什么？

**Kernel** 是在 GPU 上运行的函数，用 `__global__` 标记。

正常推理框架的工作方式：

```
for 每一层:
    CPU 调用 GPU kernel（计算这一层）
    等待 GPU 完成
    CPU 决定下一层怎么算
    CPU 调用下一个 GPU kernel
    ...
```

每次"调用 GPU kernel"都有延迟（几十微秒）。对于 24 层的模型，生成一个 token 就有约 100 次这样的调用（每层大约有 attention + MLP 等多个 kernel）。

**Megakernel 的方式**：只调用 GPU 一次，在 GPU 内部循环处理所有 24 层：

```
CPU 调用 GPU megakernel（一次）
GPU 内部:
    for layer in [0..23]:
        计算这一层
        grid.sync()  # GPU 内部同步，不回 CPU
GPU 返回结果
```

## 3. 持久化 Kernel（Persistent Kernel）

普通 kernel：调用一次，运行，结束，SM 释放。  
**持久化 kernel**：block 驻留在 SM 上，用循环处理多个 token，不退出。

Megakernel 就是持久化 kernel：

```cpp
__global__ void decode_kernel(...) {
    // 这个 kernel 在 SM 上持续运行
    // 层与层之间用 grid.sync() 同步
    for (int layer = 0; layer < 24; layer++) {
        // 处理第 layer 层
        grid.sync();  // 等所有 block 都完成本层
    }
}
```

### 持久化 kernel 的关键约束

必须确保 **所有 block 都真正在 SM 上驻留**，否则 `grid.sync()` 会死锁（等待一个根本没在运行的 block）。

RTX 3090 的 82 个 SM，每个 SM 可以驻留 1 个 BLOCK_SIZE=512 的 block，因此最多 **82 个 block** 可以安全使用 `grid.sync()`。

`setup.py` 在编译时自动检测：

```python
num_blocks = _int_env("MEGAKERNEL_NUM_BLOCKS", 82)  # 默认 82
```

## 4. BF16 和 FP16（半精度浮点数）

正常的浮点数（FP32）用 32 位表示一个数，精度高但占空间多。  
半精度只用 **16 位**，精度低一些，但：
- 显存占用减半
- 计算速度更快（GPU 有专门的半精度张量核）

两种半精度格式：
- **FP16**：数值范围小（最大约 65536），适合较老的 GPU（Turing sm_75）
- **BF16**：数值范围和 FP32 一样大，只是精度低，更适合深度学习（Ampere sm_80+）

Megakernel 的选择（`half_type.h`）：

```cpp
#if TARGET_SM >= 80
    // Ampere+ 用 BF16
    typedef __nv_bfloat16 half_t;
    #define H2F(x) __bfloat162float(x)
    #define F2H(x) __float2bfloat16(x)
#else
    // Turing 用 FP16
    typedef __half half_t;
    #define H2F(x) __half2float(x)
    #define F2H(x) __float2half(x)
#endif
```

关键：权重和激活用 BF16（节省显存/带宽），**累加时用 FP32**（保证数值精度）。

## 5. Transformer 的基本结构

Transformer 语言模型由若干层组成，每层包含两个子模块：

### 注意力子模块（Attention）

目的：让模型"注意"到序列中哪些位置的信息相关。

**标准全注意力（Full Attention）**：
- 对序列中每个 token，计算它和所有其他 token 的相关度（softmax(QK^T/√d)·V）
- 复杂度 O(n²)：序列越长，越慢

**线性注意力（DeltaNet）**：
- 用一个固定大小的"状态矩阵"（State）来记忆历史信息
- 每次只更新这个状态，不需要保存所有历史 K/V
- 复杂度 O(n)：序列越长，速度**不变**

### MLP 子模块（前馈网络）

标准的三层网络：gate × up → SiLU 激活 → down

```
hidden → gate_proj → SiLU(gate) × up_proj → down_proj → 加回 residual
```

### RMSNorm

在每个子模块前做归一化，让数值稳定：

```
输出 = 输入 / sqrt(mean(输入²)) × 权重
```

## 6. Qwen3.5-0.8B 的混合架构

Qwen3.5-0.8B 不是纯 Transformer，而是 **混合架构**：

```python
LAYER_TYPE = [0,0,0,1, 0,0,0,1, 0,0,0,1, 0,0,0,1, 0,0,0,1, 0,0,0,1]
# 0 = DeltaNet 层（线性注意力）
# 1 = Full Attention 层（全注意力）
# 规律：3个DeltaNet + 1个Full Attention，共6组
```

- 总层数：24
- DeltaNet 层：18 层（占 75%）
- Full Attention 层：6 层（占 25%）

**为什么这样设计？**  
DeltaNet 处理长序列效率高（线性复杂度），Full Attention 理解能力强（能做精确的全局关注）。混合使用，兼顾速度和效果。

## 7. KV Cache 是什么？

在解码（decode）阶段，每生成一个新 token，都需要把新 token 的 K、V 向量加入历史。

KV Cache 就是把历史所有位置的 K 和 V 存下来，避免重复计算。

Full Attention 层需要 KV Cache（因为要和所有历史 token 做注意力）。  
DeltaNet 层不需要 KV Cache（状态矩阵 State 已经压缩了历史信息）。

## 8. 共享内存（Shared Memory）

GPU 的内存层次：
- **全局内存（Global Memory）**：显存，几 GB，慢（几百 ns）
- **共享内存（Shared Memory）**：每个 SM 上的片上缓存，几十 KB，**极快**（几 ns）
- **寄存器（Registers）**：每个线程私有，最快

Megakernel 的技巧：把当前层的激活值放在共享内存里，避免频繁读写全局显存。

例如 RMSNorm 的输出先写到共享内存 `shmem`，然后矩阵乘法直接从共享内存读取输入。

---

## 9. Warp 内的通信（warp shuffle）

同一 warp 的 32 个线程之间可以直接交换寄存器值，不需要经过共享内存。

```cpp
// 将 val 在 warp 内做归约求和
float warp_reduce_sum(float val) {
    for (int offset = 16; offset > 0; offset /= 2)
        val += __shfl_down_sync(0xffffffff, val, offset);
    return val;  // thread 0 得到所有线程的总和
}
```

这是 Megakernel 里大量用到的技巧，用于 RMSNorm 的均方、矩阵乘法的点积归约等。

## 总结

| 概念 | 一句话 |
|------|--------|
| GPU Kernel | 在 GPU 上运行的函数，以 Grid/Block/Thread 形式并行 |
| Persistent Kernel | 不退出 GPU、内部循环处理、层间用 grid.sync() |
| BF16 | 16位浮点，节省带宽，累加用FP32保精度 |
| DeltaNet | 线性复杂度注意力，用状态矩阵替代KV Cache |
| Full Attention | 标准二次注意力，用KV Cache缓存历史 |
| Shared Memory | 片上极速缓存，暂存当前层的激活值 |
| Warp Shuffle | warp内32线程直接交换寄存器值，极快 |
