# Lucebox Hub 项目总览

## 一句话说明

Lucebox Hub 是一个开源项目，目标是：**在你家里的消费级显卡上，高效运行大语言模型（LLM）**。

他们的核心理念是：问题不在硬件，而在软件。一块 2020 年的 RTX 3090，用正确的方式写代码，可以媲美苹果最新芯片的能效。

---

## 为什么要做这个？

### 现有框架的问题

当你用 llama.cpp 或 PyTorch 跑一个 24 层的语言模型，生成每一个新 token 的过程大致如下：

```
CPU → 发指令启动第1层kernel → GPU跑完第1层 → 回到CPU → 发指令启动第2层kernel → ...
```

每次 "CPU → GPU → CPU" 的往返都有开销：
- CPU 发指令有延迟
- GPU 要重新从显存读取权重
- 线程需要重新同步

对于一个 24 层的模型，每生成一个 token 就需要约 **100 次 kernel 启动**。这 100 次往返累积起来，就是白白浪费的时间和电能。

### Lucebox 的解法

把全部 24 层都塞进**一个 CUDA kernel** 里，一次性跑完。层与层之间用 GPU 内部的同步机制（`grid.sync()`），完全不需要回到 CPU。

这就是 **Megakernel（巨内核）**：一次 GPU 调度，完成整个模型的前向计算。

## 三个子项目简介

### 01 · Megakernel（本文重点）

**对象模型**：Qwen 3.5-0.8B（一个 0.8B 参数的小模型）

**关键数字**：
| 方法 | 预填充 | 解码 | 能效 |
|------|--------|------|------|
| Megakernel @220W | 21,347 tok/s | 413 tok/s | 1.87 tok/J |
| llama.cpp @350W | 11,247 tok/s | 267 tok/s | 0.76 tok/J |

Megakernel 在 **用更少的电** 的情况下，速度还快了 1.5~2 倍。

**为什么选 Qwen3.5-0.8B？**  
这个模型是 **混合架构**：18 层 DeltaNet（线性注意力）+ 6 层全注意力（Full Attention），以 3:1 的比例交替排列。这种混合架构在当时没有任何框架的专门优化，是一片空白。

---

### 02 · DFlash

**对象模型**：Qwen3.5/3.6-27B（27B 参数的大模型，压缩格式）

**技术**：推测解码（Speculative Decoding）  
用一个小 draft 模型先猜多个 token，再让大模型一次性验证，从而减少大模型的调用次数。

**成绩**：在 RTX 3090 上跑 Qwen3.5-27B，解码速度提升 **3.43×**。

---

### 03 · PFlash

**对象模型**：与 DFlash 配合使用

**技术**：推测预填充（Speculative Prefill）  
用小 draft 模型判断长 prompt 中哪些 token"不重要"，让大模型只处理重要部分，大幅缩短首 token 时间（TTFT）。

**成绩**：128K 上下文的 TTFT 加速 **10.4×**。
