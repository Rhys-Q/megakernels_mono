## 1. 主程序 (main.cu)

负责解析命令行参数，示例：

```bash
nvLstmInf --precision=16 --algo=SingleStepGemv lstm_s ./data/lstm_s ./data/lstm_s.npy 30 ./output
```

| 参数 | 说明 | 可选值 |
|------|------|--------|
| `--precision` | 浮点精度 | 16（FP16）、32（FP32） |
| `--algo` | 推理算法 | `SingleStep`, `SingleStepGemv`, `AsSingleStepGemv` |
| `--num-instances` | 并发实例数 | 1-N |
| `--cpuset` | 绑定的 CPU 核心 | `0,1,2,3` |
| `--gdrcopy` | 启用 GDRCopy | 开关标志 |
| `--ping-pong` | 使用 PingPong 基准 | 开关标志 |
| `model` | 模型名称 | `lstm_s`, `lstm_m`, `lstm_l` |
| `model_dir` | 权重目录 | 路径 |
| `data_file` | 输入数据 | `.npy` 文件路径 |
| `seconds` | 运行时长 | 秒数 |

## 2. 基准测试 (Benchmark)

`Benchmark` 类负责：

1. **初始化 CUDA 环境**：检测 GPU 架构、共享内存大小等
2. **创建模型实例**：根据配置创建 1 到 N 个 `ModelInstance`
3. **分配 NUMA 内存**：在多 NUMA 节点系统上绑定内存，减少跨节点访问
4. **运行测试**：启动所有实例的推理循环，直到达到设定时长
5. **收集结果**：汇总所有实例的延迟数据

```cpp
// 伪代码示意
class Benchmark {
    void Run() {
        for (auto& inst : instances_) {
            threads_.push_back(std::thread([&inst] {
                inst.RunBenchmark(duration_);
            }));
        }
        for (auto& t : threads_) t.join();
    }
    
    void PrintResults() {
        // 打印每个实例的 P99 延迟、平均延迟等
    }
};
```

## 3. 模型实例 (ModelInstance)

这是推理循环的核心，每个 `ModelInstance` 在自己的线程中独立运行：

```cpp
void ModelInstance::RunBenchmark(float seconds) {
    model_->PreProcess();  // 预处理（持久化内核：预加载权重并等待就绪）
    
    while (运行时间 < seconds) {
        // 1. 获取下一个输入
        auto input = data_iterator_.Next();
        
        // 2. 复制输入到 GPU
        model_->Input(input);
        
        // 3. 记录开始时间
        start_time = now();
        
        // 4. 触发推理
        model_->LstmForward();
        
        // 5. 等待结果
        model_->OutputWait();
        
        // 6. 获取输出
        auto output = model_->Output();
        
        // 7. 记录结束时间并存储结果
        end_time = now();
        result_.Add(output, start_time, end_time);
    }
}
```

## 4. 模型定义 (ModelDef)

负责从磁盘加载 LSTM 模型权重。权重以 NumPy 格式（`.npy` 文件）存储：

```
data/lstm_s/
├── lstm_0_W.npy   # 第0层输入权重矩阵 (4h × d)
├── lstm_0_R.npy   # 第0层循环权重矩阵 (4h × h)
├── lstm_0_B.npy   # 第0层偏置向量 (4h)
├── lstm_1_W.npy   # 第1层输入权重矩阵
├── lstm_1_R.npy   # 第1层循环权重矩阵
├── lstm_1_B.npy   # 第1层偏置向量
├── mul.npy         # 聚合层权重向量 (h)
└── add.npy         # 聚合层偏置（标量）
```

**注意**：文件中权重的 gate 顺序是 **iofc**，而 ONNX 标准用 **iofc**，PyTorch 用 **ifco**。加载 PyTorch 导出的权重时，`convert_model.py` 会自动重排。

`ModelDef` 的主要功能：
- `LoadFromDir(path)`：从目录加载所有权重文件
- `FlopCount()`：计算模型的浮点运算量
- 提供每层权重矩阵的维度信息

## 6. 模型接口 (ModelInterface)

纯虚基类，定义所有模型实现必须遵守的接口：

```cpp
class ModelInterface {
public:
    // 将新输入从主机内存复制到 GPU
    virtual void Input(const void* host_input) = 0;
    
    // 触发 LSTM 前向推理（对持久化内核：发送"开始"信号）
    virtual void LstmForward() = 0;
    
    // 非阻塞检查推理是否完成
    virtual bool OutputReady() = 0;
    
    // 阻塞等待推理完成
    virtual void OutputWait() = 0;
    
    // 将输出从 GPU 复制回主机（对持久化内核：读取原子信号值）
    virtual float Output() = 0;
    
    // 最终聚合（点积+偏置），持久化内核中此步骤已融合
    virtual void Aggregate() = 0;
    
    // 预处理（持久化内核：启动内核并等待就绪）
    virtual void PreProcess() = 0;
    
    // 查询属性
    virtual int BatchSize() = 0;
    virtual int NumSteps() = 0;
    virtual int NumFeatures() = 0;
    virtual int NumUnits() = 0;
    virtual int Precision() = 0;  // 16 或 32
};
```

## 7. cuBLAS 模型 (ModelSingleStep)

默认的推理实现，使用 NVIDIA cuBLAS 库做矩阵运算。

### 工作流程

```
每次推理：
  
  对每一层 l = 0, 1, ..., L-1:
  
    1. 移位状态：hx[l] = [0, hx[l][0], hx[l][1], ..., hx[l][s-2]]
               cx[l] = [0, cx[l][0], cx[l][1], ..., cx[l][s-2]]
    
    2. 预计算（利用上一步的输出）：
       b_t = cublasSgemv(U[l], hx[l][-1]) + B[l]
    
    3. 等待新输入到达
    
    4. 主计算：
       gates = cublasSgemv(W[l], x) + b_t
    
    5. 逐元素 LSTM 激活：
       [i, o, f, c'] = split(gates, 4)
       cx[l][-1] = σ(f) * cx[l][-2] + σ(i) * tanh(c')
       hx[l][-1] = σ(o) * tanh(cx[l][-1])
    
    6. 下一层的输入 = 当前层的输出：x = hx[l][-1]
  
  7. 聚合：output = dot(mul_weights, hx[L-1][-1]) + add_bias
```

### 输出格式

`ModelSingleStep` 的输出是最后一层的隐藏状态向量（`h` 维），然后由框架单独执行聚合步骤得到标量输出。

## 8. 持久化模型 (ModelSingleStepGemv / ModelAsSingleStepGemv)

这是本项目最核心的技术，实现超低延迟推理。

### 两种持久化模式

**ModelSingleStepGemv**：
- 每次推理迭代，kernel 处理完成后还需重新启动
- 但权重已缓存在 shared memory 中，重启只需很短时间
- 适合中等延迟要求

**ModelAsSingleStepGemv**（更激进）：
- kernel 在基准测试开始前一次性启动
- 整个测试期间 kernel 一直在 GPU 上运行（自旋等待）
- 没有任何 kernel 启动开销
- `lstm_s` 和 `lstm_m` 使用这种模式
- `lstm_l` 由于内存限制，暂时使用每次重启模式

### 三种 kernel 变体

根据模型大小，使用不同的 kernel 架构：

| 模型 | Kernel 类型 | 文件 | 并行单元 |
|------|------------|------|---------|
| lstm_s | Block-based | `gemv_lstm_bp_impl.cu` | 单个 CUDA block |
| lstm_m | Thread-Block-Cluster | `gemv_lstm_tbc_impl.cu` | 5 或 10 个 block 的集群 |
| lstm_l | Device-wide | `gemv_lstm_dev_impl.cu` | 几乎整个 GPU |

### 输出格式

持久化模型的输出是一个**单精度浮点标量**，聚合步骤已融合在 kernel 内部。

## 9. CUDA 环境 (CudaEnv)

管理所有 CUDA 资源，是其他组件的基础设施：

```cpp
class CudaEnv {
    // CUDA 绿色上下文（GPU 资源分区）
    GreenContext green_ctx_;
    
    // CUDA 流（用于异步操作）
    cudaStream_t stream_high_priority_;  // 高优先级流（推理用）
    cudaStream_t stream_low_priority_;   // 低优先级流（预计算用）
    
    // cuBLAS 句柄
    cublasHandle_t cublas_handle_;
    cublasLtHandle_t cublaslt_handle_;
    
    // 静态查询方法
    static int MaxDynamicSharedMemory();  // GPU 最大动态共享内存
    static bool NativeAtomicSupported();  // 是否支持原生原子操作
    static int SmArch();                  // SM 架构版本（如 90 = Hopper）
};
```

**绿色上下文（Green Context）**是 CUDA 12.0+ 新增的特性：
- 把 GPU 的 SM 资源划分成多个"分区"
- 每个分区有固定的 SM 数量，不同分区相互隔离
- 多个推理实例可以在同一 GPU 上真正并行运行，互不干扰

## 11. GEMV 内核实现 (ss_gemv_impl/)

这是整个系统最底层的计算代码，运行在 GPU 上。

### 目录结构

```
ss_gemv_impl/
├── kernel_params.h      # 内核参数结构体（权重指针、同步信号等）
├── kernel_traits.h      # 编译期常量（线程数、向量化宽度等）
├── gemv_lstm.h          # 内核启动接口
├── gemv_lstm_utils.cuh  # GPU/CPU 同步原语、数学函数
├── gemv_lstm_bp_impl.cu # lstm_s 的单 block 内核
├── gemv_lstm_tbc_impl.cu # lstm_m 的 TBC 内核
├── gemv_lstm_dev_impl.cu # lstm_l 的设备级内核
└── dispatch_utils.cuh   # 分发辅助工具
```

### 内核参数结构体

内核在启动时接收一个参数结构体，包含所有需要的指针和配置：

```cpp
// 基础参数（所有内核共用）
struct Lstm_params_base {
    void* gmem_input;         // 输入向量的 GPU 内存地址
    void* gmem_h_out;         // 输出的 GPU 内存地址
    void* gmem_dot_prod_weights; // 聚合权重的 GPU 内存地址
    int*  host_ld_complete;   // CPU 可见的完成信号（pinned memory）
    int   always_spin;        // 是否使用永久自旋模式
    int   use_gdrcopy;        // 是否启用 GDRCopy
    int   max_smem_size;      // 最大共享内存大小
    int   sm_arch;            // SM 架构版本
    int   sm_count;           // SM 数量
};

// 扩展参数（每层一组指针）
template <int L, int NUM_INPUTS, int NUM_UNITS>
struct Lstm_bpt_params : public Lstm_params_base {
    void* gmem_weights[L];  // 每层权重矩阵
    void* gmem_biases[L];   // 每层偏置向量
    void* gmem_c_in[L];     // 每层输入细胞状态
    void* gmem_c_out[L];    // 每层输出细胞状态
    void* gmem_gemm_h[L];   // 每层隐藏状态
    int*  gmem_start;       // GPU 侧的开始信号
    int*  gmem_ld_complete; // GPU 侧的完成信号
};
```

### 内核执行流程（自旋模式）

```
GPU 内核主循环：

1. 所有线程加载权重矩阵到共享内存（只做一次）
2. 通知 CPU "权重已就绪"（gpu_cpu_sync_part_0）

3. 无限循环：
   a. 等待 CPU 发出"开始"信号（gpu_cpu_sync_part_1）
      - Grace Hopper：原子 compare-exchange
      - PCIe：轮询加载 + 切换开关值
   
   b. 执行 LSTM 前向计算：
      对每一层：
        - GEMV：gates = W × x + b_precomputed
        - 逐元素激活：更新 c, h
        - 传递隐藏状态给下一层
   
   c. 计算最终聚合：output = dot(mul_weights, h_last)
   
   d. 把输出值写入 CPU 可见信号（gpu_cpu_sync_part_2）
      这个写操作既是输出，也是"完成"信号
   
   e. 回到步骤 a，等待下一次推理
```

### 数学函数实现

在 `gemv_lstm_utils.cuh` 中定义了 GPU 上高效的激活函数：

```cuda
// FP32 sigmoid
static __forceinline__ __device__ float sigma_g(float x) {
    return __fdividef(1.f, 1.f + __expf(-x));  // 使用快速数学
}

// FP32 tanh
static __forceinline__ __device__ float sigma_c(float x) {
    return tanhf(x);
}

// FP16 sigmoid（更高效）
static __forceinline__ __device__ __half sigma_g_16bit(__half x) {
    return hrcp(CUDART_ONE_FP16 + hexp(-x));
}

// LSTM 逐元素计算（融合操作）
// 输入 acc = [i, o, f, c']（iofc 顺序）
// 输出更新后的 cx, hx
template <Data_kind DATA_KIND>
__forceinline__ __device__ void lstm_eltwise(float& hx, float& cx, ...) {
    // 分离 4 个门
    float4 gate = to_float4(acc);
    gate.x = sigma_g(gate.x);  // i gate (input)
    gate.y = sigma_g(gate.y);  // o gate (output)
    gate.z = sigma_c(gate.z);  // c' (cell candidate)
    gate.w = sigma_g(gate.w);  // f gate (forget)
    
    // 更新细胞状态
    cx = gate.y * c_in + gate.x * gate.z;
    
    // 更新隐藏状态
    hx = gate.w * sigma_c(cx);
}
```

> **注意**：这里的 gate 顺序是 **iofc**（i=x, o=y, f=w, c'=z），与常见文献的 **ifco** 不同，需要特别注意。
