# dl-lowlat-infer 项目概览

## 这个项目是什么？

**dl-lowlat-infer**（Deep Learning Low-Latency Inference）是 NVIDIA 开发的一个专为**超低延迟 LSTM 推理**而设计的 CUDA 加速库。

它的目标只有一个：让 LSTM（长短期记忆网络）的每次推理延迟尽可能短，达到**微秒（µs）级别**。

## 为什么需要它？

### 背景

在很多实时应用（比如金融交易、实时音频处理、工业控制）中，AI 模型每次做出预测所需的时间（延迟）是极其关键的指标。

普通的神经网络推理框架（如 PyTorch、TensorFlow）通常为了通用性和吞吐量做了优化，**没有针对单步超低延迟做特殊优化**。

### 核心挑战

LSTM 推理（单步模式）面临的核心挑战是：

1. **计算量极小**：单步 LSTM 只是一个矩阵-向量乘法（GEMV），浮点运算量极少（例如 lstm_s 只有 0.172 MFLOPS）
2. **GPU 利用率低**：计算量太小，GPU 大部分时间都在等待
3. **启动延迟大**：每次 CUDA kernel 启动本身就有几微秒的开销
4. **同步开销大**：CPU 和 GPU 之间的数据传输和信号同步非常耗时

### 解决方案

本项目通过以下核心技术解决这些问题：

- **持久化内核（Persistent Kernel）**：kernel 一次启动，永远不退出，消除重复启动开销
- **原子信号同步**：CPU/GPU 之间通过原子操作直接通信，延迟极低
- **权重预加载**：模型权重提前加载到 GPU 共享内存/寄存器中，推理时直接使用
- **GDRCopy**：利用 GPUDirect RDMA 技术让 CPU 直接读写 GPU 内存
- **绿色上下文（Green Context）**：CUDA 12.0+ 特性，允许多个推理实例共享 GPU

## 支持的模型

项目预定义了三种规模的 LSTM 模型：

| 模型 | 输入维度 | 隐藏维度 | 层数 | 时间步 | 计算量 |
|------|---------|---------|------|--------|--------|
| lstm_s（小） | 128 | 96 | 2 | 64 | 0.172 MFLOPS |
| lstm_m（中） | 192 | 160 | 3 | 96 | 0.655 MFLOPS |
| lstm_l（大） | 512 | 736 | 4 | 128 | 16.02 MFLOPS |

## 支持的硬件

- **Hopper 架构**：H100、GH200（sm_90a）
- **Blackwell 架构**：B100/B200（sm_100a, sm_120a）
- **Grace Hopper**：GH200 NVLink-C2C（ARM CPU + Hopper GPU，统一内存）
- **PCIe 系统**：普通 x86 服务器 + NVIDIA GPU
