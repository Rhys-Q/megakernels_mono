# LSTM 原理

> 如果你完全不了解 LSTM，可以先把它理解为一种有"记忆"的神经网络层。普通神经网络处理每个输入时是独立的，而 LSTM 能记住之前看到的信息，非常适合处理时序数据（音频、金融序列等）。

## 基本概念

### 变量定义

| 符号 | 含义 | 举例（lstm_s） |
|------|------|--------------|
| `d`  | 输入向量的维度 | 128 |
| `h`  | 隐藏状态（输出）的维度 | 96 |
| `l`  | 层数 | 2 |
| `x_t` | 当前时刻的输入向量，大小为 `d` | 128维向量 |
| `h_t` | 当前时刻的隐藏状态，大小为 `h` | 96维向量 |
| `c_t` | 当前时刻的细胞状态，大小为 `h` | 96维向量 |
| `W`  | 输入权重矩阵，大小为 `4h × d` | 384×128 矩阵 |
| `U`  | 循环权重矩阵，大小为 `4h × h` | 384×96 矩阵 |
| `b`  | 偏置向量，大小为 `4h` | 384维向量 |

### 激活函数

- **σ（sigmoid）**：把输入压缩到 (0, 1) 之间，用于控制"门"的开关程度
- **tanh**：把输入压缩到 (-1, 1) 之间，用于产生候选值

## LSTM 核心公式

### 第一步：计算所有门

```
gates_t = W × x_t + U × h_{t-1} + b
```

这里 `gates_t` 是一个 `4h` 维向量，把它**平均切成 4 份**，每份大小为 `h`：

```
[i_t, o_t, f_t, c'_t] = split(gates_t, 4)
```

> **注意 gate 顺序**：本项目使用 **iofc** 顺序（input, output, forget, cell），
> 这与 PyTorch 的 **ifco** 顺序不同。加载 PyTorch 模型权重时需要重新排列。

这 4 个门的含义：

| 门 | 符号 | 作用 |
|----|------|------|
| 输入门 | `i_t` | 控制有多少新信息进入记忆 |
| 遗忘门 | `f_t` | 控制遗忘多少旧记忆 |
| 输出门 | `o_t` | 控制输出多少当前记忆 |
| 候选值 | `c'_t` | 新的候选记忆内容 |

### 第二步：更新细胞状态（记忆）

```
c_t = σ(f_t) × c_{t-1} + σ(i_t) × tanh(c'_t)
```

**直觉理解**：
- `σ(f_t) × c_{t-1}`：旧记忆乘以遗忘系数（部分保留）
- `σ(i_t) × tanh(c'_t)`：新信息乘以输入系数（部分加入）
- 两者相加就是新的记忆内容

### 第三步：计算隐藏状态（输出）

```
h_t = σ(o_t) × tanh(c_t)
```

**直觉理解**：从当前记忆 `c_t` 中提取信息，用输出门 `o_t` 控制输出多少。

## 滑动窗口推理模式

本项目使用**滑动窗口（Sliding Window）**模式进行推理：

```
时间步:   t-3  t-2  t-1   t
窗口1:   [  0,   1,   2,   3 ]
窗口2:        [  1,   2,   3,   4 ]
窗口3:             [  2,   3,   4,   5 ]
```

每来一个新的输入 `x_t`，就做一次单步推理，输出一个结果。

**状态管理**：每次推理后，隐藏状态 `h_t` 和细胞状态 `c_t` 会向前滑动一位：
```
旧状态: [s_0, s_1, s_2, ..., s_{n-1}]
新状态: [s_1, s_2, s_3, ..., s_{n-1}, s_new]
```

其中 `s_new` 是最新时刻计算出的状态，而第一个位置在非流式模式下会重置为 0。

## 快速推理优化：预计算技巧

注意完整公式是：
```
gates_t = W × x_t + U × h_{t-1} + b
```

其中 `U × h_{t-1} + b` 可以在等待下一个输入时**预先计算**：

```python
# 预计算（利用等待输入的空闲时间）
b_t = U × h_{t-1} + b

# 真正来了输入之后，只需做：
gates_t = W × x_t + b_t  # 比完整计算快！
```

这样把一次大矩阵乘法拆成两次，利用时间差做了并行流水。

## 最终聚合

多层 LSTM 处理完后，最后一层的隐藏状态 `h_t` 还要经过一个聚合步骤才能得到最终的标量输出：

```
output = dot(mat_mul_weights, h_t_last_layer) + bias
```

也就是把 `h` 维向量点乘一个权重向量，加上偏置，得到一个浮点数。

## 计算量分析

LSTM 的主要计算量来自矩阵-向量乘法（GEMV）。总 FLOPS 公式为：

```
总 FLOPS = 2 × (4 × h × d + (l-1) × 4 × h × h)
```

以 lstm_l 为例：
- 第1层：`4 × 736 × 512 = 1,507,328` 次乘加
- 第2-4层（每层）：`4 × 736 × 736 = 2,166,784` 次乘加
- 总计：约 16 MFLOPS

这个数字放在现代 GPU（峰值 TFLOPS 级别）面前极其微小，所以**瓶颈不在计算，而在延迟**。
